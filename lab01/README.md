# Лабораторная работа №1

## Предметная область
   Паттерн «Строитель» используется в системах автоматизированного проектирования (САПР) и конструкторах для создания объектов со сложной внутренней структурой. Один из примеров — программный комплекс для моделирования частных жилых домов.  
**Конструктор домов** – это инструмент, позволяющий пользователю поэтапно проектировать здание, выбирая базовые параметры (размеры, материалы) и дополняя их специфическими модулями (бассейн, гараж, сад), в зависимости от выбранного архитектурного решения.
## Описание проблемы
   Процесс создания дома может варьироваться от простейшей постройки до элитного особняка. Если возлагать логику сборки на клиентское приложение, то код будет перегружен условными операторами для проверки каждого флага («нужен ли гараж?», «нужен ли бассейн?»). Это приводит к «раздуванию» кода и дублированию логики при создании разных типов домов.

## Решение проблемы
   Использование паттерна «Строитель». Он отделяет алгоритм конструирования (последовательность шагов) от конкретной реализации этих шагов.

- Директор задает общую схему: «Заложить фундамент, возвести стены, добавить опции».

- Строитель реализует эти шаги для конкретного типа дома (например, PoorHouseBuilder проигнорирует этап «добавить опции», а FancyHouseBuilder установит все доступные модули).

- Клиент получает готовый продукт, не вникая в детали того, какие именно свойства объекта были изменены на каждом этапе.
